The WEKA Open Source Retrieval-Augmented Generation (RAG) project is designed to combine the power of large language models (LLMs) with high-performance data retrieval, providing accurate, context-aware, and verifiable responses. At its core, RAG integrates two major processes — retrieval and generation. The retrieval component efficiently indexes and searches through large volumes of text or vectorized data (using tools such as FAISS), while the generation component uses a local or cloud-based LLM (e.g., Llama or similar) to produce natural language answers informed by the retrieved context

Installation Details

This blueprint requires that the WEKA Operator/CSI driver has been installed and configured on your targeted Kubernetes cluster. A Storage Class for the WEKA CSI driver needs to be setup prior to the installation and selected in the Storage Class configuration. This blueprint uses ReadWriteMany Persistant Volumes. Using Storage Classes not capiable of working within this PVC requirenment will fail to install

This blueprint uses models hosted and downloaded from HuggingFace. This blueprint assumes the user already has a HuggingFace account with an API key. If this is not the case please go to HuggingFace website and enroll with an account and create an API key.

Once a HuggingFace API key has been created please add that key into the settings page into the HuggingFace section, please make sure when adding the HuggingFace key the correct Kubernetes namespace has been selected to create the secret. This blueprint requires a Kubernetes secret with the HuggingFace API key within the namespace this blueprint will be deployed into.

Some Large Language Models (LLM) from HuggingFace require you to select an EULA prior to downloading. Please check in the HuggingFace website against the LLM models you wish to use and agree to the EULA prior to deploying this blueprint.
