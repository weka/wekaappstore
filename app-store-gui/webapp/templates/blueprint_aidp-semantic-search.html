Introduction

This add-on deploys Open WebUI as a single sign-on chat interface in front of your NeuralMesh AIDP deployment. Users log in with their existing Keycloak identity and ask questions in natural language; answers are grounded in your own documents through the AIDP Retrieval-Augmented Generation (RAG) pipeline, complete with source citations.

It is an add-on — it reuses the AIDP platform you already deployed (Keycloak identity, the RAG gateway, and the NIM language model) rather than standing up its own. Configuration is automatic: the only value you provide is the external URL the UI will be reachable on.

What this blueprint installs

  • Open WebUI — the chat UI (official Helm chart), with persistence on WEKA storage.
  • Keycloak OIDC client — an open-webui client registered in the existing rag-gateway realm, with Keycloak group → role mapping (admins / users).
  • AIDP RAG Search tool — a built-in tool (aidp_rag_bridge) that queries your knowledge base through the AIDP rag-bridge.
  • “AIDP RAG Assistant” model — a public workspace model wired to the AIDP NIM LLM (nvidia/llama-3.3-nemotron-super-49b-v1.5) with the RAG Search tool attached by default.
  • Ingress route — an Envoy HTTPRoute publishing the UI on your chosen URL.

How it works

  • Prerequisite: the NeuralMesh AIDP blueprint must already be installed and Ready in the same namespace.
  • No credentials needed: NVIDIA NGC and WEKA credentials were already materialized by AIDP; this add-on reuses them.
  • Single sign-on: users authenticate against the existing Keycloak realm; members of the admins / cluster-admins groups get Open WebUI admin.
  • Grounded answers: questions route through the RAG Search tool to rag-bridge → the RAG pipeline, returning answers with citations from your own data.
  • Auto-configured: the OIDC provider URL, client secret, and internal routing are derived at deploy time from AIDP’s existing config — you only supply the external URL below.
