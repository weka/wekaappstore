  <name>Task 1: Fix blueprints.py logger TypeError and wire LOG_LEVEL</name>
  <files>mcp-server/tools/blueprints.py, mcp-server/server.py, mcp-server/tests/test_blueprints.py, mcp-server/tests/test_logging.py</files>
  <action>
1. In `mcp-server/tools/blueprints.py` line 60, remove the `file=sys.stderr` kwarg from the `logger.warning()` call. The `logging` module does not accept `file=` — that is a `print()` kwarg. The corrected line should be:
   ```python
   logger.warning("Failed to parse %s: %s", yaml_file, exc)
   ```
   Also remove the `import sys` at line 12 since it is no longer used in this file (verify no other usage first).

2. In `mcp-server/server.py`, change the hardcoded `level=logging.INFO` on line 15 to use the config value:
   ```python
   import config  # noqa: E402 — needed before basicConfig for LOG_LEVEL
   ```
   BUT config must be imported BEFORE `logging.basicConfig()` is called. Since config.py only uses `os.environ.get()` at module level (no FastMCP dependency), it is safe to import before FastMCP. Move the import of config above the `logging.basicConfig()` call, then use:
   ```python
   logging.basicConfig(
       stream=sys.stderr,
       level=getattr(logging, config.LOG_LEVEL, logging.INFO),
       format="%(asctime)s %(name)s %(levelname)s %(message)s",
   )
   ```

3. In `mcp-server/tests/test_blueprints.py`, add a test that creates a temporary directory with one malformed YAML file and one valid WekaAppStore YAML file, calls `scan_blueprints()`, and asserts:
   - The result contains only the valid blueprint (malformed was skipped)
   - No exception was raised
   Name: `test_scan_blueprints_skips_malformed_yaml`

4. In `mcp-server/tests/test_logging.py`, add a test that patches `os.environ` with `LOG_LEVEL=DEBUG`, reimports or reloads config, and asserts `config.LOG_LEVEL == "DEBUG"`. This confirms the env var is read. (The wiring to server.py is verified by inspection since basicConfig runs at import time.)
  </action>
