<task type="auto">
  <name>Task 1: Apply manifests to EKS cluster and wait for pod readiness</name>
  <files>scripts/validate-phase13.sh</files>
  <action>
Apply all Phase 13 manifests to the live EKS cluster in the correct order:

1. Apply RBAC first (SA must exist before pod references it):
   ```
   kubectl apply -f k8s/agent-sandbox/mcp-rbac.yaml -n wekaappstore
   ```

2. Apply SKILL.md ConfigMap:
   ```
   kubectl apply -f k8s/agent-sandbox/mcp-skill-configmap.yaml -n wekaappstore
   ```

3. Apply updated Sandbox CR (this triggers pod recreation with new containers):
   ```
   kubectl apply -f k8s/agent-sandbox/openclaw-sandbox.yaml -n wekaappstore
   ```

4. Wait for the pod to be Ready (all containers running):
   ```
   kubectl wait --for=condition=Ready pod -l <sandbox-selector> -n wekaappstore --timeout=300s
   ```
   Use the sandbox selector resolution pattern from validate-topology.sh.

5. If the pod does not become Ready within 300s, gather diagnostics:
   - `kubectl describe pod <pod> -n wekaappstore` (check Events for pull/crash errors)
   - `kubectl logs <pod> -c openclaw-json-generator -n wekaappstore` (init container logs)
   - `kubectl logs <pod> -c weka-mcp-sidecar -n wekaappstore` (sidecar startup logs)
   - `kubectl logs <pod> -c git-sync -n wekaappstore` (sync errors)

6. Run the live validation script:
   ```
   bash scripts/validate-phase13.sh --live wekaappstore
   ```

If live validation reveals issues, diagnose and fix the manifests. Common issues to check:
- RBAC 403: ClusterRoleBinding namespace mismatch (fix subjects[0].namespace)
- Init container failure: printf syntax error (check init container logs)
- MCP sidecar CrashLoopBackOff: missing env var or wrong image tag
- git-sync failure: repo URL or permissions (check git-sync logs)
- SKILL.md not found: ConfigMap name mismatch or volume not mounted
  </action>

  </verify>
  <done>All manifests applied. Pod is Ready with all containers running. validate-phase13.sh --live passes all checks (RBAC, health endpoint, openclaw.json content, SKILL.md mount, blueprint sync).</done>

Verification steps:

1. Check pod status shows all containers running:
   ```
   kubectl get pods -n wekaappstore -l agents.x-k8s.io/sandbox-name-hash
   ```
   Expected: READY shows 3/3 (openclaw, weka-mcp-sidecar, git-sync)

2. Check MCP sidecar health:
   ```
   kubectl exec openclaw-sandbox -n wekaappstore -c weka-mcp-sidecar -- curl -sf http://localhost:8080/health
   ```
   Expected: HTTP 200

3. Check openclaw.json was generated correctly:
   ```
   kubectl exec openclaw-sandbox -n wekaappstore -c openclaw -- cat /home/node/.openclaw/openclaw.json
   ```
   Expected: JSON with transport=streamable-http, url=http://localhost:8080/mcp

4. Check SKILL.md is mounted:
   ```
   kubectl exec openclaw-sandbox -n wekaappstore -c openclaw -- head -5 /home/node/.openclaw/SKILL.md
   ```
   Expected: First 5 lines of SKILL.md content

5. Check blueprints are syncing:
   ```
   kubectl exec openclaw-sandbox -n wekaappstore -c weka-mcp-sidecar -- ls /app/blueprints/
   ```

6. Check OpenClaw logs for MCP server discovery:
   ```
   kubectl logs openclaw-sandbox -n wekaappstore -c openclaw | grep -i "mcp\|tool\|weka"
   ```
