# Phase 25 — UI Design Contract

> Visual and interaction contract for the Blueprint Credential Selector SDK. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** A tiny surface — two Jinja2 macros (`credential_select`, `weka_storage_select`) inside `app-store-gui/webapp/templates/_credential_macros.html`. This spec inherits the existing WEKA App Store visual language from `settings.html` and `blueprint.html` and locks the exact Tailwind classes, focus states, empty-state copy, and the `warpSyncEndpoint` interaction. **No new tokens, palettes, fonts, or components are introduced.**

## Spacing Scale

Declared values (must be multiples of 4). The macros use only Tailwind utilities that map to this scale — no custom values, no exceptions.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px (`mb-1`, `gap-1`) | Label-to-input gap |
| sm | 8px (`mb-2`, `gap-2`, `py-2`) | Input vertical padding; select vertical padding |
| md | 12px (`px-3`) | Input horizontal padding |
| lg | 16px (`mt-4`, `space-y-4`) | Vertical rhythm between form fields |
| xl | 24px (`p-6`) | Card padding (inherited from `.card.rounded-lg.p-6` in `blueprint_neuralmesh-aidp.html:308`) |

**Exceptions:** none.

**Label-to-input spacing:** label `block` + `mb-1` (4px), matching `settings.html:686`.
**Field-to-field spacing:** `space-y-4` on the wrapper (16px), matching `settings.html:684`.
**Hint paragraph spacing (empty state):** `mt-2` (8px) below absent select position.

## Color

Inherits the existing 60 / 30 / 10 split from `settings.html` and `blueprint.html`. The macros introduce no new colors.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0b0c10` (body), `#e5e7eb` (body text) | Page background and value text inside the macro |
| Secondary (30%) | `rgba(31, 41, 55, 0.6)` (`.card`); `bg-gray-800/70` for input/select surfaces | Card surface (supplied by host); input/select fill (`settings.html:688`) |
| Accent (10%) — WEKA purple `#6b2fb3` | reserved for: **focus ring on input/select** (`focus:ring-2 focus:ring-[var(--weka-purple)]`) AND **link underline color in the empty-state hint** (`text-[var(--weka-purple)] underline` — matches `settings.html:138`) | Nothing else inside the macro uses purple |

**Accent reserved for (explicit list):**
1. The focus ring on `<select>` (both macros)
2. The focus ring on `<input type="url">` (`weka_storage_select` only)
3. The `<a href="/settings#credentials">` link inside the empty-state hint (`credential_select` only)

No accent on the `<select>` resting border, no accent on the label, no accent on the input fill.

**Borders:** all form controls use `border border-white/10` (matches `settings.html:688`). No coloured borders in the resting state.

## Component Contract — Exact HTML Output

### Macro 1 — `credential_select(type, field_name, label=None, required=True)`

**Two output shapes**, decided by whether `credentials_by_type[type]` has any entries.

#### A. Populated state (one or more ready credentials)

```html
<div class="space-y-1">
  <label for="cred-{{ field_name }}" class="block text-xs font-medium text-white/70 mb-1">
    {{ label or type | replace('-', ' ') | title }}
  </label>
  <select
    id="cred-{{ field_name }}"
    name="{{ field_name }}"
    {% if required %}required{% endif %}
    class="w-full px-3 py-2 rounded-md bg-gray-800/70 border border-white/10 focus:outline-none focus:ring-2 focus:ring-[var(--weka-purple)] text-sm">
    {% for cred in credentials_by_type[type] %}
      <option value="{{ cred.name }}">{{ cred.displayName }}</option>
    {% endfor %}
  </select>
</div>
```

#### B. Empty state (no ready credentials of this type)

```html
<div class="space-y-1">
  <label class="block text-xs font-medium text-white/70 mb-1">
    {{ label or type | replace('-', ' ') | title }}
  </label>
  <p class="text-sm muted">
    No {{ type | replace('-', ' ') }} credential configured.
    <a href="/settings#credentials" class="text-[var(--weka-purple)] underline hover:text-white">Add one in Settings.</a>
  </p>
</div>
```

> The empty-state shape, copy, and link color match the existing pattern at `settings.html:136-140` ("No WEKA Storage credential configured. Add one above."). This is the single canonical empty-state hint pattern in this codebase — reuse it verbatim.

### Macro 2 — `weka_storage_select(credential_field, endpoint_field, label)`

**Two output shapes** — same populated/empty branch as Macro 1, but the populated branch also renders an endpoint URL input and an inline `<script>` for the `warpSyncEndpoint` interaction.

#### A. Populated state

```html
<div class="space-y-4">
  <div class="space-y-1">
    <label for="cred-{{ credential_field }}" class="block text-xs font-medium text-white/70 mb-1">
      {{ label }}
    </label>
    <select
      id="cred-{{ credential_field }}"
      name="{{ credential_field }}"
      required
      onchange="warpSyncEndpoint(this)"
      class="w-full px-3 py-2 rounded-md bg-gray-800/70 border border-white/10 focus:outline-none focus:ring-2 focus:ring-[var(--weka-purple)] text-sm">
      {% for cred in credentials_by_type['weka-storage'] %}
        <option value="{{ cred.name }}" data-endpoint="{{ cred.endpoint or '' }}">{{ cred.displayName }}</option>
      {% endfor %}
    </select>
  </div>
  <div class="space-y-1">
    <label for="ep-{{ endpoint_field }}" class="block text-xs font-medium text-white/70 mb-1">
      {{ label }} Endpoint
    </label>
    <input
      id="ep-{{ endpoint_field }}"
      name="{{ endpoint_field }}"
      type="url"
      required
      value="{{ credentials_by_type['weka-storage'][0].endpoint or '' }}"
      class="w-full px-3 py-2 rounded-md bg-gray-800/70 border border-white/10 focus:outline-none focus:ring-2 focus:ring-[var(--weka-purple)] text-sm" />
  </div>
</div>
<script>
  function warpSyncEndpoint(selectEl) {
    var opt = selectEl.options[selectEl.selectedIndex];
    var epId = 'ep-' + selectEl.name.replace(/^.*$/, '{{ endpoint_field }}');
    var epInput = document.getElementById(epId);
    if (epInput && opt) { epInput.value = opt.dataset.endpoint || ''; }
  }
</script>
```

#### B. Empty state

```html
<div class="space-y-1">
  <label class="block text-xs font-medium text-white/70 mb-1">{{ label }}</label>
  <p class="text-sm muted">
    No WEKA Storage credential configured.
    <a href="/settings#credentials" class="text-[var(--weka-purple)] underline hover:text-white">Add one in Settings.</a>
  </p>
</div>
```

## Interaction Contract — `warpSyncEndpoint`

| Property | Value |
|----------|-------|
| Trigger | `<select>` `onchange` event (declared inline as `onchange="warpSyncEndpoint(this)"`) |
| Read source | `selectEl.options[selectEl.selectedIndex].dataset.endpoint` |
| Write target | `<input type="url">` with `id="ep-{{ endpoint_field }}"` |
| Empty-endpoint behaviour | If `data-endpoint` is empty or absent, write `''` (clear the field) — never leave a stale value |
| Initial value | The endpoint input is pre-populated server-side from `credentials_by_type['weka-storage'][0].endpoint` (the first option's endpoint), matching the default `<select>` selection |
| User editability | The endpoint input remains user-editable after `warpSyncEndpoint` writes to it — no `readonly` attribute |
| Script scope | Inline `<script>` lives inside the macro output. If the macro is called twice on the same page, only the latest function definition wins — acceptable because both definitions resolve to the same behaviour against different DOM ids |

**Focus order (tab navigation):** label is not focusable; user tabs from credential `<select>` → endpoint `<input>` → next form field. Both controls share the identical purple focus ring, so the affordance is consistent.

## Form States (matrix)

| State | `<select>` appearance | `<input type="url">` appearance | Notes |
|-------|----------------------|--------------------------------|-------|
| Resting | `bg-gray-800/70` + `border-white/10` | same | Inherits from `settings.html:688` |
| Focused | `ring-2 ring-[var(--weka-purple)]`, `outline-none` | same | Purple focus ring is the only accent affordance |
| Disabled | not used in this phase — macros never render a disabled select | — | If no credentials exist, the empty-state hint replaces the select entirely (Macro 1) or the entire block (Macro 2) |
| Invalid (HTML5) | browser-default red glow on `:invalid` after submit attempt | same | The `required` attribute is the only validation contract; no custom error styling is introduced in this phase |
| Empty state | not rendered — replaced by hint `<p>` | not rendered | See Macro output shape B above |

**Error state copy is intentionally out of scope** for the macros — the deploy form's existing `<div id="deploy-result">` (at `blueprint_neuralmesh-aidp.html:315`) owns submit-time error messaging. The macros do not write to it.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Not owned by this phase — the Configure card's existing `Deploy Blueprint` button (`blueprint_neuralmesh-aidp.html:314`) is unchanged |
| `credential_select` default label (when `label=None`) | Title-cased credential type, e.g., `Nvidia Ngc`, `Huggingface`. Template authors should always pass an explicit `label` for human-readable text (e.g., `"NVIDIA NGC API Key"`) |
| `credential_select` empty-state body | `No {type with hyphens replaced by spaces} credential configured. Add one in Settings.` |
| `weka_storage_select` empty-state body | `No WEKA Storage credential configured. Add one in Settings.` (verbatim match to `settings.html:137`) |
| Empty-state link text | `Add one in Settings.` — link target `/settings#credentials` |
| Endpoint field label suffix | `{{ label }} Endpoint` — e.g., `WEKA NeuralMesh Endpoint` |
| Destructive confirmation | Not applicable — the macros render no destructive controls |
| Error state (submit-time) | Not owned — handled by host form's existing `#deploy-result` element |

**Copy guarantees (locked):**
- The empty-state link is the exact deep-link `/settings#credentials` (never `/settings` alone).
- Linked text reads `Add one in Settings.` — period included, sentence-case, no exclamation.
- No emoji, no icon character (e.g., no `⚠`) in the empty-state hint — it matches the muted, plain-text tone of `settings.html:136-140`.

## Reference Example — `blueprint_neuralmesh-aidp.html` Configure Card

The plan must update the Configure card (`blueprint_neuralmesh-aidp.html:308-322`) to insert both macro calls between the hidden namespace input (line 312) and the Deploy button (line 314).

**Expected insertion (decisions D-09):**

```jinja
{% from '_credential_macros.html' import credential_select, weka_storage_select %}

<div class="card rounded-lg p-6">
  <h2 class="text-lg font-semibold mb-3">Configure</h2>
  <form id="deploy-form" class="space-y-4">
    <input type="hidden" id="namespace" name="namespace" value="default" />

    {{ weka_storage_select(credential_field="weka_credential", endpoint_field="weka_endpoint", label="WEKA NeuralMesh") }}
    {{ credential_select(type="nvidia-ngc", field_name="ngc_credential", label="NVIDIA NGC API Key") }}

    <button type="submit" class="btn-purple w-full px-4 py-2 rounded-md text-sm font-medium">Deploy Blueprint</button>
    <div id="deploy-result" class="text-xs mt-2"></div>
  </form>
  ...
</div>
```

**Width contract:** because each macro's outermost wrapper has no explicit width (only `space-y-*`), each control naturally fills the form column at `w-full`. The Configure card lives in the right-rail column (`<div class="space-y-6">` at `blueprint_neuralmesh-aidp.html:~298`), so the controls render at the card's interior width (`p-6` interior of a `.card` in a `lg:grid-cols-3` column ≈ 320–360px). No `max-w-*` cap is needed.

## Verifiable Acceptance Criteria

The executor MUST produce a `_credential_macros.html` that satisfies all of the following — these are the auditor's checks:

1. `credential_select` populated state renders exactly one `<label>` + one `<select>` with classes `w-full px-3 py-2 rounded-md bg-gray-800/70 border border-white/10 focus:outline-none focus:ring-2 focus:ring-[var(--weka-purple)] text-sm` (verbatim match to `settings.html:688`).
2. `credential_select` empty state renders a `<p class="text-sm muted">` containing the hint text and an `<a href="/settings#credentials" class="text-[var(--weka-purple)] underline hover:text-white">Add one in Settings.</a>`.
3. `weka_storage_select` populated state renders a credential `<select>` whose every `<option>` carries a `data-endpoint` attribute, plus a sibling `<input type="url">` with the same input classes as item 1.
4. `weka_storage_select` includes an inline `<script>` block defining `warpSyncEndpoint(selectEl)` that copies `selectEl.options[selectEl.selectedIndex].dataset.endpoint` to the endpoint `<input>`'s `value`.
5. The credential `<select>` element has `onchange="warpSyncEndpoint(this)"` wired in its tag (not bound via JS after render).
6. No new CSS custom property is introduced; no `<style>` block is added in `_credential_macros.html`.
7. No new font is loaded; no script tag other than the inline `warpSyncEndpoint` definition appears in the macro.
8. The empty-state link target is exactly `/settings#credentials` — not `/settings`, not `/settings/credentials`.
