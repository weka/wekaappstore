Create directory `scripts/`.

**scripts/install-agent-sandbox.sh** -- Installs agent-sandbox operator v0.2.1:
- `#!/usr/bin/env bash` with `set -euo pipefail`
- Set `AGENT_SANDBOX_VERSION="v0.2.1"` at top
- Step 1: `kubectl apply -f` the manifest.yaml and extensions.yaml from the GitHub releases URL
- Step 2: `kubectl wait --for=condition=Ready pods --all -n agent-sandbox-system --timeout=120s`
- Step 3: Verify CRD exists: `kubectl get crd sandboxes.agents.x-k8s.io` (fail if not found)
- Step 4: Print success message with controller pod status
- Make executable: `chmod +x`

**scripts/validate-topology.sh** -- Validates Phase 12 topology (based on 12-RESEARCH.md smoke test):
- `#!/usr/bin/env bash` with `set -euo pipefail`
- Accept namespace as `$1` with default from env var `NAMESPACE` or fallback to `default`
- `SANDBOX_NAME="openclaw-sandbox"`
- 5 validation steps:
  1. Pod Running: `kubectl wait --for=condition=Ready` on pod with label `sandbox.agents.x-k8s.io/name=$SANDBOX_NAME` (300s timeout)
  2. GPU allocated: Get pod, check `resources.limits.nvidia.com/gpu` equals "1"
  3. OpenClaw gateway: `kubectl exec` into pod and `curl -sf --max-time 10 http://localhost:18789/healthz` -- WARN on failure (not FAIL, since healthz path is unconfirmed per open question #1 in research)
  4. GPU node: Get the pod's nodeName, verify `nvidia.com/gpu` allocatable >= 1
  5. Loopback port 8080 path probe (WARN-only): `kubectl exec` into pod and `curl -sf --max-time 5 http://localhost:8080/ 2>&1`. This is a WARN-only step -- no MCP sidecar listener exists yet (Phase 13), so the expected result is "Connection refused" (exit code 7). This proves the loopback network path to port 8080 is OPEN even though nothing is listening. If curl returns a different error (e.g., timeout, which would indicate a NetworkPolicy block), print a WARNING that the loopback path may be blocked. Specifically:
    - Exit code 7 (connection refused) = PASS (path open, no listener -- expected)
    - Exit code 0 (unexpected success) = PASS (something is listening -- fine)
    - Exit code 28 (timeout) = WARN: "Loopback path to :8080 may be blocked by NetworkPolicy"
    - Any other error = WARN with the error details
    - This step NEVER causes a FAIL -- it is informational for NCLAW-03 evidence
- Print summary: pod name, node name, GPU count, loopback 8080 status
- Exit 0 on pass, exit 1 on any FAIL (loopback probe WARNs do not cause FAIL)
- Make executable: `chmod +x`
