    function resolveNamespaceSync() {

|----------|-------------|
| BLUEPRINTS_DIR filesystem → parse_x_variables | Blueprint YAML files authored externally (git-synced); their content is parsed with yaml.safe_load — safe_load is explicitly sandboxed against arbitrary Python object construction. |
| client → FastAPI route (`GET /blueprint/{name}`) | Path parameter `name` is now passed to `find_blueprint` which performs an `os.walk` scan. The walk is bounded to `BLUEPRINTS_DIR` — no path traversal possible because `os.walk` only descends into subdirectories of the root, not parent directories. |
| variable_schema dict → blueprint.html Jinja2 | Keys and values from blueprint YAML `x-variables` are rendered into HTML. Jinja2 auto-escapes string output by default in FastAPI's Jinja2Templates, preventing XSS from malicious blueprint YAML content. |
