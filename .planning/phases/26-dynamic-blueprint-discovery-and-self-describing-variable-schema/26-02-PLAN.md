<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP query param `variables` → deploy_stream JSON parse | User-supplied JSON string parsed with `json.loads`. Malformed JSON yields an SSE error event — no server crash. |
| Parsed user_vars dict → template.render(**user_vars) | Variable values from user input are passed to Jinja2 `[[var]]` template render. Jinja2 with custom delimiters does not execute arbitrary Python — only replaces `[[var_name]]` tokens. |
| Blueprint YAML content → schema validation | The schema (from parse_x_variables) governs which vars are required. Blueprint authors control the schema — a malicious blueprint could mark all vars as non-required, bypassing validation. This is accepted: blueprint authors already have write access to BLUEPRINTS_DIR. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-05 | T (Tampering) | `variables` JSON query param — key injection | mitigate | `template.render(**user_vars)` passes only the keys present in user_vars. Jinja2's `[[var]]` syntax with custom delimiters only substitutes declared tokens; unknown keys are silently ignored by Jinja2 (they do not cause errors and cannot inject new YAML keys). |
| T-26-06 | I (Information Disclosure) | Error messages naming missing variables | accept | The SSE error `"Required variable missing: storage_class"` exposes the variable name but not its value. Variable names come from the blueprint schema (already visible to the user who is filling the form) — no sensitive information is disclosed. |
| T-26-07 | T (Tampering) | `apply_blueprint_content_with_namespace` called with user-rendered YAML | accept | The rendered YAML is applied to Kubernetes. A user who can reach the deploy endpoint already has access to submit blueprints. The WEKA App Store is an operator-level tool, not a multi-tenant public service. The existing apply function is unchanged — its security posture is identical to the pre-Phase-26 codebase. |
| T-26-08 | D (Denial of Service) | Large `variables` JSON string | mitigate | `json.loads` is called on a query parameter string; FastAPI/uvicorn apply HTTP request size limits (default 1MB) before the handler runs. No additional guard needed. |
</threat_model>
