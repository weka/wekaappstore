        - message: string

    - wekaEndpoint: type: string (optional; only populated for weka-storage credentials)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CR author → Kubernetes API server | WarpCredential CR submitted by cluster administrators; admission validation is the gate |
| Helm chart → cluster | Chart rendered by Helm and applied by a cluster admin; no user-supplied data flows into the template at render time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-01 | Tampering | spec.type field | mitigate | Enum constraint in CRD schema rejects any value not in [nvidia-ngc, huggingface, weka-storage] at admission — no runtime check needed |
| T-21-02 | Information Disclosure | status.wekaEndpoint | accept | Endpoint URL is non-sensitive (no credentials); surfacing it in status is intentional per PRD design |
| T-21-03 | Elevation of Privilege | CRD scope | accept | CRD is Namespaced — CRs are scoped per namespace; no cluster-wide credential exposure |
</threat_model>
