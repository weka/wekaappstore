component, the `EmptyState` component, filter state, URL hash synchronization (both directions,

above the catalog grid, clicking a card filters the grid in place, the URL hash mirrors the
selection, and keyboard + screen-reader users get equivalent functionality via native `<button>`
toggle semantics with `aria-pressed`.

This plan encodes all five critical pitfalls from PITFALLS.md as explicit implementation rules
and verification checks:

1. **JSX forbidden** — every element uses `h()`; `<[A-Z]` grep inside the script block must
   return zero matches (Pitfall 3).
2. **`component: 'a'` forbidden on category `CardActionArea`** — category cards must render as
   `<button>`, not `<a>` (Pitfall 4).
3. **`history.replaceState` must NOT fire on mount** — initial hash is read inside a lazy
   `useState` initializer; mount writes nothing to history (Pitfall 2).
4. **Hash parser uses `startsWith('#category=')` + enum validation** against
   `['aidp', 'warp', 'partner']` to avoid collisions with existing `#catalog` /
   `#planning-studio` scroll anchors (Pitfall 1).
5. **`ThemeProvider` stays lifted in `AppShell`** — this plan does not re-add it anywhere else
   (Pitfall 5 — already mitigated structurally by 15-02; this plan must not regress it).

This plan MUST NOT:
- Revert any 15-02 structural change (Catalog prop-based, AppShell wraps ThemeProvider, mount
  id is `app-root`).
- Add a new CDN script, change the MUI destructure list beyond what already exists, or modify
  the MUI theme.
- Add a count `Chip` on each category card — per REQUIREMENTS.md the CAT-04 live count is

- Use `pushState`, set `location.hash = ...`, or call `replaceState` from any location other

- Introduce auto-scroll on deep-link mount (Gray Area D = no auto-scroll).

**Hash parser contract (for PITFALL 1 defense):**

```javascript
function parseCategoryHash() {
  const raw = window.location.hash;                    // "", "#catalog", "#category=warp", etc.
  if (!raw.startsWith('#category=')) return 'all';     // STRICT prefix check
  const key = raw.slice('#category='.length);
  const valid = ['aidp', 'warp', 'partner'];           // enum validation
  return valid.includes(key) ? key : 'all';
}
```

**Hash write contract (for PITFALL 2 defense):**

```javascript
// Lives inside useEffect([selectedCategory]) — runs AFTER mount.
// The lazy initializer above is read-only: it sets state from the hash but never writes.
useEffect(() => {
  if (selectedCategory === 'all') {
    history.replaceState(null, '', window.location.pathname);          // strips hash
  } else {
    history.replaceState(null, '', '#category=' + selectedCategory);   // uses replaceState, NOT pushState
  }
}, [selectedCategory]);
```

On first mount, React runs the lazy initializer → sets `selectedCategory` from hash → renders →
then runs effects. The effect fires on mount with the just-initialized value and calls
`replaceState` with the SAME hash that was already in the URL. Per PITFALL 2, `replaceState`
with an identical URL is not a no-op on the back stack — it replaces the browser's original
entry for the page, which is the bug we're defending against. Mitigation: guard the effect
against the "mount-with-same-hash" case by storing a ref or by comparing the current URL
before calling `replaceState`. Recommended guard:

```javascript
useEffect(() => {
  const target = selectedCategory === 'all'
    ? window.location.pathname
    : window.location.pathname + '#category=' + selectedCategory;
  const current = window.location.pathname + window.location.hash;
  if (current !== target) {
    history.replaceState(null, '', target);
  }
}, [selectedCategory]);
```

This way:
- Mount with `/#category=warp` → initializer reads `'warp'` → effect fires, current === target, no write. Back stack intact.
- Mount with `/` (no hash) → initializer reads `'all'` → effect fires, current === target, no write.
- User click on WARP when in All → state becomes `'warp'` → effect writes `/#category=warp`. Replaces current entry. Back stack: single entry.
- User click WARP again (toggle off) → state becomes `'all'` → effect writes `/` (no hash). Replaces current entry. Back stack: single entry.

Key result (URL-02): no matter how many category toggles, the back stack has ONE entry for
the home page. Pressing Back exactly once leaves the site.
