<objective>
Keep secret values out of the persisted CR annotation and the SSE stream. Define ONE secret-key predicate, use it both to build an annotation allowlist (`_safe_gui_variables`) and to build the SSE redactor's secret-value-set, so the two redaction points share a single definition. Closes ROADMAP Phase 29 SC4 (SEC-01, E2E-03).

Purpose: The annotation at ~2935 is persisted in etcd and visible via `kubectl get wekaappstore -o yaml`; the SSE component messages are operator-supplied and may echo manifest content into the browser/proxy logs. Both are secret-leak vectors. The single-definition requirement prevents the predicate and the value-set from diverging (a known incomplete-redaction failure mode).
Output: a shared secret-key predicate, `_safe_gui_variables` (drops secret keys), an SSE message redactor (replaces actual secret VALUES with `***`), wiring at both sites, and unit tests proving no secret leaks at either point.
</objective>
