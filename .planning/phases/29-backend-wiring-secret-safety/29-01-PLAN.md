<objective>
Wire the backend so `app-store-install` is treated as a namespace-preserving blueprint everywhere `cluster-init` is special-cased, and replace the hardcoded 900s SSE deadline with a per-blueprint `x-deploy-timeout` (raised default when absent). Closes ROADMAP Phase 29 SC1 (namespace preserve) and SC3/PROG-02 (raised deadline).

Purpose: The Phase 27 `app-store-install.yaml` has fixed per-component `targetNamespace` values and `metadata.namespace: default`; a user-selected namespace must NOT overwrite them. The full operator+CSI+WekaClient install runs ~5 sequential stages each with `timeout: 300` readinessChecks, so the legacy 900s cap false-fails a healthy install.
Output: `NAMESPACE_PRESERVING_APPS` module-level set, a `parse_deploy_timeout` helper, an `x-deploy-timeout` key in the blueprint, and unit tests proving namespace-preserve + timeout behavior.
</objective>
