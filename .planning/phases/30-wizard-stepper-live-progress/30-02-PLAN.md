<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser ↔ /deploy-stream (GET query string) | EventSource is GET-only; the `variables` JSON (including quay_password, weka_password) travels in the URL query string |
| /deploy-stream SSE ↔ browser DOM | Per-component and completion messages stream back into the progress view |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-05 | Information Disclosure | Secrets (quay_password, weka_password) passed as `/deploy-stream` query-string parameters because EventSource only supports GET | accept (documented) | EventSource cannot send a body, so `variables` JSON goes in the URL — this exactly matches the existing battle-tested blueprint.html pattern (lines 279-284). The connection is same-origin to the App Store backend the customer already trusts. Residual risk: secrets may appear in browser history and server access logs. Accepted for v8.0 because (a) it is the established pattern, (b) the App Store is an authenticated internal admin tool, (c) changing to POST would require a new backend route which is out of scope (frontend-only phase). Flag for a future hardening item (move to POST + token-fetch pattern). |
| T-30-06 | Information Disclosure | Secret values surfaced in the streamed SSE messages displayed inline (D-07) | mitigate | complete.message / component.message are secret-redacted server-side by _redact_secrets (main.py:3116/3126, Phase 29). The client displays them verbatim only because they are already redacted — no client-side secret rendering. Task 2 displays installError (the redacted message) directly. |
| T-30-07 | Information Disclosure | Secrets persisted client-side across the install/retry cycle | mitigate | buildVariables() reads from in-memory React `form` state only (Plan 01 D-02). Retry re-reads the same in-memory state; nothing is written to localStorage. No new persistence introduced. |
| T-30-08 | Denial of Service | A failed install leaves a dangling open EventSource | mitigate | Every terminal branch (complete, error, onerror) calls es.close(); Retry opens a fresh stream. No leaked connections. |
| T-30-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase; no new CDN dependencies. N/A. |
