<objective>
Implement both Phase 28 code changes inside the single operator module:
- OPA-02: remove `@lru_cache` from `discover_chart_crds` and replace it with a module-level `_chart_crds_cache` dict that is written ONLY when `helm show crds` succeeds, so an auth/network failure is never memoized as "no CRDs."
- OPA-01: thread an optional `registry_config_path` parameter through `discover_chart_crds`, `should_skip_crds_for_component`, and `HelmOperator.install_or_upgrade` / `_install_chart` / `_upgrade_chart`; in `handle_appstack_deployment`, write the quay docker auth JSON to a temp file once per OCI helmChart component (guarded by `oci://` + `quay_dockerconfigjson` present), pass `--registry-config <path>` to `helm show crds` / `install` / `upgrade`, and delete the file in a `try/finally`.

Purpose: De-risk the highest-risk install path — a fresh-cluster operator must authenticate to quay before OCI chart pulls and must not permanently cache an empty-CRD failure (which would later 404 the WekaClient apply).
Output: Modified `operator_module/main.py` (the ONLY file this phase touches, per D-08).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator pod → quay.io OCI registry | Operator helm process presents quay docker auth to pull/install the OCI operator chart. Auth material originates from `appStack.variables.quay_dockerconfigjson`. |
| operator process → subprocess argv / logs | Helm is invoked via `subprocess.run`/`check_output` with explicit arg lists; argv and DEBUG logs are observable artifacts. |
| operator process → local filesystem (temp dir) | Temp registry-config file holds quay docker auth on disk for the duration of one OCI component's helm operations. |
