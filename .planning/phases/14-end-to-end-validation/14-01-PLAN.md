Create `scripts/validate-phase14-prereqs.sh` — a bash script that validates all preconditions before the E2E chat session:

1. Accept namespace as first argument (default: wekaappstore)
2. Run `bash scripts/validate-phase13.sh $NAMESPACE --live` and check exit code — if FAIL, abort with message "Phase 13 preconditions not met"
3. Resolve pod name dynamically: `SELECTOR=$(kubectl get sandbox openclaw-sandbox -n $NAMESPACE -o jsonpath='{.status.selector}')` then `POD=$(kubectl get pods -n $NAMESPACE -l "$SELECTOR" -o jsonpath='{.items[0].metadata.name}')`
4. Check OSS Rag blueprint exists: `kubectl exec $POD -c weka-mcp-sidecar -n $NAMESPACE -- ls /app/blueprints/` and grep for a directory or file containing "oss" or "rag" (case-insensitive). If not found, WARN (not FAIL) — blueprint naming may differ.
5. Check all 3 containers are Ready: parse `kubectl get pod $POD -n $NAMESPACE -o jsonpath='{.status.containerStatuses[*].ready}'` — all must be "true"
6. Check MCP health endpoint: `kubectl exec $POD -c weka-mcp-sidecar -n $NAMESPACE -- python3 -c "import urllib.request; r=urllib.request.urlopen('http://localhost:8080/health'); print(r.status)"` — must output 200
7. Print summary: "Phase 14 Prerequisites: X/Y PASS, Z WARN" with instructions to start port-forward: `kubectl port-forward pod/$POD 18789:18789 -n $NAMESPACE`
8. Exit 0 on all PASS, exit 1 on any FAIL, exit 0 with warnings on WARN-only

Use the same output formatting style as validate-phase13.sh (check numbers, PASS/FAIL/WARN labels, colored output if terminal supports it).

Create `scripts/capture-e2e-evidence.sh` — a bash script that captures kubectl evidence for all 4 E2E requirements:

1. Accept namespace as first argument (default: wekaappstore)
2. Create `evidence/` directory if it doesn't exist

   - `kubectl get nodes -o wide > evidence/e2e-01-nodes.txt`

4. Capture E2E-02 evidence:
   - Resolve pod dynamically (same pattern as prereqs script)

   - `kubectl get pods -n $NAMESPACE -o wide > evidence/e2e-04-status.txt`
   - `kubectl describe wekaappstore -n $NAMESPACE >> evidence/e2e-04-status.txt` (describes all CRs)
7. Print: "Evidence captured in evidence/ directory" with file listing
8. Add a `--pre` flag that only captures E2E-01 and E2E-02 (before the chat session) and a `--post` flag that captures E2E-03 and E2E-04 (after the chat session). No flag captures all.

Both scripts must be executable (chmod +x).
