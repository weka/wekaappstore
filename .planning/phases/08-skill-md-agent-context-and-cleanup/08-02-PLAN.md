    2. Extracts tool names and descriptions from the registered tools (use mcp.list_tools() or inspect the registered tool metadata - check FastMCP API)
    3. Builds the openclaw.json structure per CONTEXT.md decisions:
       - name: "weka-app-store-mcp"
       - description: brief description of the WEKA App Store MCP server
       - transport: "stdio"
       - startup: { command: "python", args: ["-m", "server"], cwd: "mcp-server/" }
       - env.required: ["BLUEPRINTS_DIR"] (required for blueprint catalog)
       - env.optional: ["KUBERNETES_AUTH_MODE", "LOG_LEVEL", "KUBECONFIG"] (from config.py)
       - container: placeholder image reference (e.g., "weka-app-store-mcp:latest")
       - tools: array of {name, description} for all 8 tools
       - Note at top-level or in a _comment field: "Best-effort format — may need revision when NemoClaw alpha schema is published"
    4. Writes the JSON to mcp-server/openclaw.json with indent=2
    5. Script is runnable standalone: `python generate_openclaw_config.py`
    6. Also prints a summary of how many tools were registered

    Then run the script to generate the initial openclaw.json.

    If FastMCP doesn't expose a clean way to list registered tools programmatically, an alternative approach: parse the @mcp.tool() decorated functions from tools/*.py to extract names and docstrings. Use whichever approach is simpler and more reliable.
