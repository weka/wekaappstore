## Removal Contract

This section is the primary deliverable of Phase 23's UI work. It specifies exactly what is removed and the no-regression expectation.

### Endpoints to Remove from `app-store-gui/webapp/main.py`

| Handler | Location | Reason |
|---------|----------|--------|
| `@app.post("/api/secret/huggingface")` ‚Äî `save_huggingface_key` | Lines 558‚Äì570 | Replaced by `POST /api/credentials` (API-03, API-07) |
| `@app.post("/api/secret/nvidia")` ‚Äî `save_nvidia_key` | Lines 626‚Äì638 | Replaced by `POST /api/credentials` (API-03, API-07) |

No other route handlers are modified. The `create_or_update_secret()` helper at lines 535‚Äì555 is retained ‚Äî it is reused by the new credentials API.

### HTML Sections to Remove from `app-store-gui/webapp/templates/settings.html`

| Section | Location | Element IDs involved |
|---------|----------|----------------------|
| HuggingFace API Key card (`<section>`) | Lines 53‚Äì81 | `hf-api-key`, `hf-namespace`, `hf-save`, `hf-result`, `hf-secrets-list`, `hf-listing-scope` |
| NVIDIA API Key card (`<section>`) | Lines 84‚Äì112 | `nvidia-api-key`, `nvidia-namespace`, `nvidia-save`, `nvidia-result`, `nvidia-secrets-list`, `nvidia-listing-scope` |

After removal, the remaining page sections are in this order:
1. Kubernetes Auth Status (was Section 3, becomes Section 1)
2. Cluster Status (was Section 4, becomes Section 2)
3. Blueprint Uninstall (was Section 5, becomes Section 3)
4. Debug (was Section 6, becomes Section 4)

### JavaScript to Remove from `settings.html` `<script>` block

| Function / Handler | Lines | Reason |
|--------------------|-------|--------|
| `getSettingsNamespace()` | 321‚Äì328 | Only read `hf-namespace` / `nvidia-namespace` DOM elements being removed |
| `setSettingsNamespace(ns)` | 330‚Äì341 | Only wrote to removed DOM elements and `localStorage.selectedNamespace` |
| `renderSecrets(containerId, items)` | 343‚Äì360 | Only rendered into `hf-secrets-list` / `nvidia-secrets-list` containers being removed |
| `loadSecrets(ns)` | 362‚Äì383 | Called `renderSecrets` and `getSettingsNamespace`; all targets removed |
| `hfBtn` click handler block | 387‚Äì412 | Posted to `/api/secret/huggingface` (endpoint removed) |
| `nvBtn` click handler block | 415‚Äì440 | Posted to `/api/secret/nvidia` (endpoint removed) |
| `loadSecrets()` call at line 485 | 485 | Initial call to removed function |

The `loadBlueprints()` call at line 570 and `refreshAuthStatus()` call at line 486 are retained ‚Äî they are unaffected.

Note: `loadBlueprints()` at line 547 calls `getSettingsNamespace()` only when `scope === 'current'`. After removal of `getSettingsNamespace`, that call site in `loadBlueprints` must be replaced with an inline fallback: `const namespace = (scope === 'current') ? (localStorage.getItem('selectedNamespace') || 'default') : 'all';`. This is required to avoid a runtime `ReferenceError`.

### No-Regression Baseline

After the removal, the settings page must:

- Load without JavaScript errors in the browser console
- Display Kubernetes Auth Status, Cluster Status, Blueprint Uninstall, and Debug sections in that order
- Continue polling `/auth-status` every 10 seconds (unchanged)
- Continue loading and deleting blueprints via `/api/blueprints` (unchanged)
- Continue persisting the Debug toggle to `localStorage` (unchanged)
- Not reference any removed DOM IDs or function names
- Return HTTP 404 for any request to `/api/secret/huggingface` or `/api/secret/nvidia` (endpoints removed)

Phase 24 (Settings GUI Overhaul) will add new Credential Management sections to replace what is removed here.
