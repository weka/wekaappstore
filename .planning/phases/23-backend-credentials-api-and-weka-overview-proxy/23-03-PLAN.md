<objective>
Add `GET /api/weka/overview?credential=<name>` to `app-store-gui/webapp/main.py`. The handler resolves the named WarpCredential of type `weka-storage`, reads its raw Secret, exchanges username + token for a Bearer token via `POST /api/v2/login`, calls `/api/v2/fileSystems`, `/api/v2/cluster`, and `/api/v2/containers` in parallel, assembles the documented response shape, caches it per credential for 60 seconds, and supports `?bust=1` for forced refresh. Covers API-05, API-06, and continues the API-08 no-leak posture.

Purpose: Phase 24 (Settings GUI) renders a WEKA Storage Overview panel populated by this endpoint. The 60-second cache prevents hammering the WEKA management plane on rapid page reloads.

Output: One new route, four new module-level helpers, one module-level cache dict. No new dependencies (`urllib.request`, `json`, `base64`, `time`, `asyncio`, `ssl` are stdlib; all currently imported per main.py header).
</objective>

- POST {endpoint}/api/v2/login
  body: {"username": <WEKA_API_USERNAME>, "password": <WEKA_API_TOKEN>}
  expected response: JSON containing an access token; the PRD describes this as a Bearer token usable on subsequent calls. The exact JSON path for the access token (e.g. `access_token`, `data.access_token`, `token`) MUST be confirmed at implementation time against the live Swagger UI at https://<cluster>:14000/api/v2/docs.
- Authorization header for subsequent calls: `Authorization: Bearer <access_token>`

Data calls (one Bearer token per overview request, three parallel calls):
- GET {endpoint}/api/v2/fileSystems  → filesystem inventory; per-item fields `name`, `uid`, total/used capacity (PRD lists candidate field names `total_budget` or `size` and `used_total` or `used_size` — verify against live Swagger before coding the assembly)
- GET {endpoint}/api/v2/cluster      → cluster-wide capacity (subfields TBD per Swagger — typically nested under a `capacity` key; verify)
- GET {endpoint}/api/v2/containers   → container list; filter where `role == "BACKEND"` (or `mode == "BACKEND"` per Swagger); IP available either inline or via /api/v2/containers/{uid}/netdevs (verify which the deployed WEKA version returns)

PRD-specified output shape (API-06):
{
  "capacity": {"totalBytes": <int>, "usedBytes": <int>, "availableBytes": <int>, "usedPercent": <float>},
  "filesystems": [{"name": <str>, "totalBytes": <int>, "usedBytes": <int>, "usedPercent": <float>}],  // no uid
