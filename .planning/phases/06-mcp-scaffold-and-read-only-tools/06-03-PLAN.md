    **mcp-server/tools/crd_schema.py:**

    `register_crd_schema(mcp)`:
    - `@mcp.tool()` on `get_crd_schema() -> dict`
    - Docstring: "Call this tool when you need to generate WekaAppStore YAML from scratch. Returns the CRD OpenAPI v3 schema defining valid fields and structure, plus 1-2 example manifests from the blueprint catalog. Use the schema to validate your YAML structure and the examples to pattern-match correct formatting. Sequencing: get_blueprint -> get_crd_schema -> (generate YAML) -> validate_yaml."
    - Read CRD via injectable `apiextensions_api.read_custom_resource_definition("wekaappstores.warp.io")` — use same injectable pattern as `collect_cluster_inspection`
    - Accept optional `apiextensions_api` parameter for testing (but tool function itself takes no MCP params — use a module-level setter or import-time injection)
    - Actually: define the tool function to call a private helper `_get_crd_schema_impl(apiextensions_api=None)` that accepts the injectable param. The tool function calls the helper with defaults. Tests call the helper directly with mocked API.
    - Extract `group`, `version` (from stored version), `kind`, and `schema` (openAPIV3Schema converted to dict via `.to_dict()` if needed, or access attributes)
    - For examples: call `scan_blueprints(config.BLUEPRINTS_DIR)`, take first 2, convert manifest dicts back to YAML strings via `yaml.dump()`
    - Catch `ApiException` (404 = CRD not installed, others = K8s unavailable) and return structured response with warnings

    Response shape:
    ```python
    {
        "captured_at": "2026-...",
        "group": "warp.io",
        "version": "v1alpha1",
        "kind": "WekaAppStore",
        "schema": { ... openAPIV3Schema dict ... },  # NOTE: this is a nested dict but it IS the schema — agent needs full depth. This is the ONE exception to 2-key depth: schema contents are pass-through from K8s, not our structure.
        "examples": ["apiVersion: warp.io/v1alpha1\nkind: WekaAppStore\n...", ...],
        "warnings": []
    }
    ```

    NOTE on depth contract: The `schema` field contains the raw CRD OpenAPI schema which is inherently deeply nested. This is pass-through data from K8s, not our domain model. The 2-key depth rule applies to OUR structural decisions (no nested wrappers). Document this exception in test_response_depth.py.
